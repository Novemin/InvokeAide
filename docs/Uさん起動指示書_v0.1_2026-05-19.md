## 2. プロジェクト概要

### 2.1 InvokeAide とは

- **コードベース名**: InvokeAide
- **プロダクト仮称**: 秘書召喚アプリ
- ユーザーが自分の右腕として AI 秘書を「召喚」し、対話を通じて日常を支援してもらう道具

- 現行 MIYU = 実験場・先行検証
- InvokeAide = 安定版・ベータ v1.0 配布対象
- 詳細は CLAUDE.md §1.4 参照

### 2.3 ベータ v1.0 配布スケジュール

- 配布日: **2026-07-04(土)または 07-05(日)** の家族集まり
- 削減判定ゲート: 2026-06-23
- 多少 7月にずれ込んでも問題なし

## 4. Uさん の担当スコープ(本指示書で確定)

Sさん が `Phase1_スケジュール現実性評価_2026-05-18.md` §1.3 で提案したスコープを、本指示書で正式採用します。

### 4.1 担当6項目

1. **VOICEVOX Cloud Run デプロイ**(完全独立)
   - Google Cloud Run へのデプロイ作業
   - 自前ホストの構築・保守
   - 月額コスト想定: 無料枠〜商品化時 月$10程度(Sさん 報告)

2. **Drive API 統合**
   - ユーザーの Google Drive にデータを保管する仕組み
   - ベータから全員 Drive 完結方針(IndexedDB 中心ではない)
   - 認証フロー、読み書きインターフェース

3. **Calendar Event 通知発火**(コーチング機能のインフラ)
   - InvokeAide ではコーチング機能を Calendar Event リマインダー方式で実装
   - 通知の発火機構を担当
   - プッシュテンプレートの内容(口調・優先順位判定)は Sさん 担当

4. **UI コンポーネント実装**
   - Vue 3 + Vite + TypeScript ベース
   - 共通コンポーネントの実装

5. **キャラクター選択UI**(MIYU + セバスチャン)
   - 初回起動時のキャラ選択画面
   - 設定からの切替機能
   - キャラ別の口調プロンプト読み込み

6. **iOS Safari 実機検証**(Sさん と並行)
   - 落とし穴集(`Phase1_iOS_Safari_落とし穴集_v0.1_2026-05-18.md`)を参照
   - 実機での動作確認

### 4.2 Sさん との分業ルール

- **Sさん = ロジック品質**: TaskCoachingContext、プッシュテンプレート、キャラ別口調プロンプト
- **Uさん = インフラ**: 発火機構、デプロイ、UI コンポーネント
- 境界が曖昧な箇所は、独断で進めずたかしさん経由でエルトンに確認

## 7. 重要原則

### 7.1 独断で進めない

着手前に不明点があれば、独断で進めず、たかしさん経由でエルトンに確認:
- 仕様の曖昧な箇所
- 優先順位の判断
- 技術選定の判断材料不足
- 想定外の事態
- Sさん との境界が曖昧な実装範囲

### 7.2 「未来の自分を縛らない」原則

実装でも法務でも、 **断定表現を避け、限定表現を使う**:

| 避ける表現 | 推奨表現 |
|---|---|
| サーバーは存在しない | 専用サーバーを運用していない |
| データを一切受信しない | 通常運用において収集・保存しない |
| 100% 安全 | 合理的なセキュリティ対策を講じる |

コードのコメント、ドキュメント、エラーメッセージ等にも適用される。

### 7.3 「対等な相棒」として意見を出す

- 「上司だから」「AIだから」を理由にした遠慮はしない
- たかしさんの判断にも「ちょっと待った」と言ってOK
- 自信のある領域では遠慮せず提案する

### 7.4 「分からない」を共有する文化

- 分からないことを「分かったふり」で進めない
- 仮の解釈で進める時は必ず明示する
- 想定外の事態は早めに共有する

### 7.5 メモリの正直な訂正

セッションメモリで誤認が判明したら、**隠さず堂々と書き直す**。
