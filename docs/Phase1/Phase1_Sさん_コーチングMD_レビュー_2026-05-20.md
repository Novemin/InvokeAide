### Q-S-11: TaskCoachingContext の JSON 添付方式 — 🟢 賛同(ただし安定性検証必要)

- **Vitest + MSW で LLM レスポンスをモック化、出力に変数(`{title}`)が残らないことを assert**

### 3.2 LINE_EXCLUDE パターン

`scripts/check-coaching-prompts.js` の `LINE_EXCLUDE` は coaching.md 内の **「使わない」リスト記述行** を検査対象外にする(自己言及の誤検知回避)。Sさん コーチングMD §5 / §6 内の NG ワード列挙(例: 「『君を見守ってる』『いつも近くにいる』『永遠に』『絶対に〜する』」)が誤検知されないことを確認:

```js
const LINE_EXCLUDE = [
  /^\s*-\s+「.*」.*$/, // 引用形式の NG ワード列挙
  /NG ワード/,
  /避ける/,
];
```

これで Sさん の coaching.md がリスト形式で NG ワードを「これを使うな」と書いている行は除外され、 **実際の「LLM 応答」 が NG ワードを含む場合のみ検出** される設計。
