#### 1.1.2 「iOS_Safari」の複合語問題(微細な論点)

命名規則統一案 §3.1 テンプレートは「`_` で区切る」ルールだが、 **「iOS Safari」は英語固有名詞の複合語** で、自然な英語表記には半角スペースが含まれる。ファイル名上で半角スペースを避けるため `_` で代用すると、区切り文字としての `_` と混同される。

可能性のある分解パターン:
- パターンA: `iOS` = プロダクト名 / `Safari` = サブ用途 / `落とし穴集` = ??
- パターンB: `iOS_Safari` = 分類(複合) / `落とし穴集` = サブ用途
- パターンC: `iOS Safari 落とし穴集` = 単一の分類

実態としてはパターンB の意図と思われる。**現状で実用上の問題はない** が、将来「PWA」「Service Worker」のような複合語が出てきた時のためにルール化を推奨。

**素案(命名規則統一案 §10 への追記候補)**:

> 複合語(英語固有名詞、半角スペースを含む語)を分類として扱う場合、`_` で接続する。例: `iOS_Safari_...`、`Service_Worker_...`。
