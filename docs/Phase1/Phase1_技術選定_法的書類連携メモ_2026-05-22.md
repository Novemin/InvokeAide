## 1. Privacy Policy 反映依頼 — 音声合成(VOICEVOX Cloud Run 中継)

### 1.1 反映依頼の根拠

- 技術選定: バックエンド層 2 = VOICEVOX セルフホスト on Google Cloud Run(v0.4 §2 / CLAUDE.md §5.1 確定)
- 構造的制約: 音声合成のため、 AI 応答テキストは Cloud Run(運営者管理下)を経由する
- ただし: 通常運用において保存しない設計(リクエスト処理 → レスポンス返却 → メモリ破棄、 アクセスログのみ記録)

### 1.2 反映観点

- (a) 音声合成のために AI 応答テキストが Cloud Run を経由する事実の開示
- (b) Cloud Run 側でテキスト本文を保存しない設計の明示
- (c) アクセスログ(レイテンシ計測用)のみ記録、 テキスト内容は記録しない の明示

### 1.3 Sさん 推奨表現例

```
本アプリの音声機能を利用される際、 AI が生成した応答テキストを、 運営者
(Novem Intelligence)が管理する音声合成サーバー(Google Cloud Run 上で稼働)
に一時的に送信します。 このサーバーは、 受け取ったテキストを音声データに
変換した上で、 通常運用においては当該テキスト本文を保存しません。 アクセス
状況の確認のためのログ(処理時刻・処理時間等)は保存される場合がありますが、
テキスト本文は含まれません。
```

「未来縛らない原則」 適用:
- ×「絶対に保存しません」 / ○「通常運用においては保存しません」
- ×「テキスト本文は一切記録しません」 / ○「アクセスログにはテキスト本文は含まれません」

### 1.4 起草担当向け確認事項(必要に応じて Sさん 経由で詳細提供可)

- VOICEVOX Cloud Run のアクセスログに含まれる項目の具体一覧
- ログ保存期間(現状未確定、 運用設計次第)
- リクエストヘッダにユーザー識別子が含まれるか(現状: Bearer Token は経由しない設計想定)

## 2. Privacy Policy 反映依頼 — 音声入力(Web Speech API)

### 2.1 反映依頼の根拠

- 技術選定: STT = Web Speech API(OS 標準音声入力)(v0.4 §4 確定)
- 構造的制約: Web Speech API は内部的に OS / ブラウザベンダー(Apple / Google 等)のクラウド STT を呼ぶ実装が多い
- 運営者(Novem Intelligence)サーバーは経由しないが、 **OS ベンダー側にデータが渡る点は明示開示が必要**

### 2.2 反映観点

- (a) 音声入力は端末標準の音声認識機能を使う旨
- (b) OS / ブラウザの仕様で音声データが OS 提供元に送信される場合がある旨
- (c) 運営者(Novem Intelligence)のサーバーには音声データそのものが送信されない旨
- (d) OS 提供元のプライバシーポリシーは各ベンダー側を参照する旨
- (e) 認識結果のテキストは Drive 内保存 or LLM API へ送信される場合がある旨(これは音声機能特有ではなく、 通常のテキスト入力と同じ扱い)

### 2.3 Sさん 推奨表現例

```
本アプリでの音声入力は、 ご利用の端末(iPhone / Android / PC)の標準的な
音声認識機能を使用します。 一部の端末・ブラウザでは、 仕様により、 音声
データが OS 提供元(Apple / Google 等)のサーバーで処理される場合があり
ます。 この場合の音声データの取り扱いについては、 各 OS 提供元の
プライバシーポリシーをご確認ください。

なお、 運営者(Novem Intelligence)のサーバーには、 音声データそのものは
送信されません。 音声から変換されたテキスト(認識結果)のみが、 必要に応じて
AI への問い合わせや、 ユーザー所有の Google Drive 内への保存に利用されます。
```

「未来縛らない原則」 適用:
- ×「音声データを一切送信しません」 / ○「運営者のサーバーには音声データそのものは送信されません」
- ×「OS ベンダーはデータを保存しません」(運営者が約束できない事項) / ○「OS 提供元のプライバシーポリシーをご確認ください」

### 2.4 起草担当向け確認事項

- iOS Safari / Android Chrome / PC ブラウザ それぞれで Web Speech API の挙動が違う点を法的に開示する粒度(包括的開示 or 個別開示)
- 「OS 提供元」 列挙の網羅性(Apple / Google / Microsoft 等、 どこまで挙げるか)

## 3. 利用規約反映依頼 — LLM(第三者 AI サービス)の抽象化

### 3.1 反映依頼の根拠

- 技術選定: LLM 抽象化 = AIProvider interface (adapter pattern)、 初期 GeminiProvider のみ実装、 Claude / OpenAI は contract 互換 stub(v0.4 §3 確定)
- 法的要件: CLAUDE.md §4.1「第三者AIサービス抽象化(Gemini 固定でない設計)」、 法的書類起草指示書 v0.3 第11章

### 3.2 反映観点

- (a) 現時点で対応している AI サービスの明示(Gemini)
- (b) 将来、 他の AI サービスへの対応を検討している旨
- (c) AI サービスは第三者(Google / Anthropic / OpenAI 等)が提供するものであり、 各社の利用規約・プライバシーポリシーに従う旨
- (d) BYOK(ユーザー自身の API キー利用)前提のため、 API 利用料金はユーザー負担である旨
- (e) AI 応答の正確性・信頼性は AI サービス提供元の挙動に依存する旨

### 3.3 Sさん 推奨表現例

```
本アプリは、 AI による応答生成のため、 第三者が提供する大規模言語モデル
(LLM)サービスを利用します。 現時点では、 Google が提供する「Gemini」 に
対応しています。 将来、 Anthropic 社の「Claude」 や OpenAI 社の「ChatGPT」
等、 他の AI サービスへの対応を検討しています。

ご利用にあたっては、 ユーザー様ご自身で対応 AI サービスの API キーを取得
していただく必要があります(BYOK: Bring Your Own Key 方式)。 API 利用に
伴う料金は、 ユーザー様と AI サービス提供元との直接の契約となり、 運営者
(Novem Intelligence)は関与しません。

AI サービスのご利用にあたっては、 各 AI サービス提供元の利用規約・
プライバシーポリシーが適用されます。 AI が生成する応答の内容について、
運営者は正確性・適切性を保証するものではありません。
```

「未来縛らない原則」 適用:
- ×「Gemini のみを利用します」 / ○「現時点では Gemini に対応しています」
- ×「Claude / OpenAI には対応しません」 / ○「将来、 他の AI サービスへの対応を検討しています」
- ×「他社モデルは絶対に追加されません」 / ○「対応サービスは将来変更される場合があります」

### 3.4 起草担当向け確認事項

- BYOK の鍵管理(SecretStore Web Crypto AES-GCM、 端末内暗号化)の説明粒度
- 「AI 応答の正確性を保証しない」 表現の法的妥当性(免責の範囲)
- 各 AI サービス提供元の利用規約への参照リンクをユーザーに提示する義務の有無

## 4. 「未来縛らない原則」 表現例集(CLAUDE.md §3.3 適用)

法的書類 v0.4 全体で使える表現対応表:

| 避ける表現 | 推奨表現 | 適用例 |
|---|---|---|
| 一切〜しない | 通常運用では〜しない | データ保存、 送信、 記録 |
| 永久に〜しない | 現時点では〜していない | 機能制限、 対応範囲 |
| 100% 安全 | 合理的なセキュリティ対策を講じる | データ保護、 暗号化 |
| 絶対に〜できない | 〜するための仕組みを設けていない | データ取得、 削除 |
| 完璧に〜する | 〜するよう努める | 法令遵守、 品質維持 |
| 必ず〜する | 〜することを基本とします / 〜する場合があります | データ削除、 通知配信 |
| 〜のみ対応 | 現時点で〜に対応 | LLM、 STT、 TTS |
| 〜には対応しない | 現時点では〜には対応していない / 将来〜を検討しています | 機能、 サービス |
| 〜は絶対しない | 通常の運用フローには含まれません | 第三者提供、 ログ収集 |

### 6.3 開示すべきが本メモで未掘削の論点(将来確認)

本メモのスコープ外だが、 法的書類 v0.4 起草中に論点化する可能性のあるもの:
- Google Drive 内に保存されるデータの暗号化粒度(現状は Drive 自体の暗号化に依存、 端末側で追加暗号化は SecretStore の API キーのみ)
- Calendar Event リマインダーの通知配信に伴うデータの取り扱い
- 多端末利用時の LWW(Last-Write-Wins)同期挙動(コンフリクト時のデータ消失リスクの説明)
- BYOK 鍵紛失時のユーザー救済範囲(運営者は鍵を保持しないため復元不可)

これらは法的書類 v0.4 起草過程で論点化された場合、 Sさん へ問い合わせください。
