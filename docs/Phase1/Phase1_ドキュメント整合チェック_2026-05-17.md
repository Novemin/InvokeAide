### 1.2 主要な仕様変更点(v1.2 → v1.4)

#### 1.2.1 ToDo 運用: 3リスト方式 → 1リスト方式(v1.3 で転換)

| 項目 | v1.2 | v1.4 |
|---|---|---|
| リスト数 | 3(未完了/完了/キャンセル) | 1(`ToDo_by_MIYU`) |
| 状態管理 | リスト間移動で表現 | ☐/☑/削除で表現 |
| ランク機能 | ON/OFF 任意、デフォルト OFF | ON 前提、A/B/C/D/P が確定 |
| 完了処理 | リスト間移動(API 上は title リネーム想定) | `complete_task` で status="completed" に(API ベース) |
| キャンセル保存 | 3ヶ月、その後アーカイブ | 削除(事前確認あり)、archive 言及なし |
| 完了保存 | 3ヶ月、その後アーカイブ | (v1.4 では言及なし、要確認) |

**整合観点での気づき**: v1.4 §17 は v1.3 改訂を継承する形式で、v1.2 §17.6/17.7 の「3ヶ月保存 → アーカイブ」というデータライフサイクル定義が v1.4 では明示的に書き直されていません。1リスト方式の下での完了/キャンセルタスクの長期保存方針が空白になっています。

### 2.2 重要齟齬の詳細

#### 🔴 齟齬1: データ取り扱い表現の断定/限定の不一致

| ドキュメント | 該当表現 |
|---|---|
| 仕様書 v1.4 §15.1(v1.2 継承) | 「たかしさん側のサーバー: **存在しない**」 |
| 仕様書 v1.4 §15.5(v1.2 継承) | 「MIYU は会話内容を **一切受信・保存しない**」「**いかなるデータにもアクセスできない**」 |
| 法的書類 v0.3 §6.5.1 | 「専用サーバーを **運用していません**」「**通常運用において、収集・保存しません**」「**アクセスする手段を持ちません**」 |
| 法的書類 v0.3 §1.2 原則 | 「断定表現を避け、限定表現を使う」 |

**素案(エルトン本文起草用)**: 仕様書 v1.4 §15 の本文を法的書類 v0.3 §6.5.1 の置換ルールに沿って書き直す。具体的には:

| v1.4 §15 現状 | 推奨書き換え |
|---|---|
| たかしさん側のサーバー: 存在しない | 運営者は、会話内容を保存・閲覧することを目的とした専用サーバーを運用していない |
| MIYU は会話内容を一切受信・保存しない | 通常運用において、運営者は会話内容を収集・保存しない |
| API キーは…外部送信されない | API キーは端末内に暗号化保存され、通常運用において運営者側に送信されない |
| MIYU 開発者は…アクセスできない | 通常運用において、運営者はユーザーの会話内容にアクセスする手段を持たない |

**影響範囲**: 仕様書 §15 全体 + §9.4 マーケティングメッセージ(後者は別判断、後述)。

#### 🔴 齟齬2: 商品化時の利用規約事項(§15.5)が法的書類と表現乖離

齟齬1 と同根。v1.2 §15.5 で「商品化時の利用規約に明記すべき事項」として列挙された4項目が、法的書類 v0.3 §6.5.1 の置換ルール対象とほぼ完全一致。仕様書側で書き換えるか、 **「法的書類 v0.3 に従って起草する」と仕様書側で参照させる** 形に変えるのも選択肢(後者の方が一元管理しやすい)。

#### 🔴 齟齬6: AI 抽象化レイヤー(第三者 AI 切替) — 仕様書が Gemini 固定の前提

| ドキュメント | 表現 |
|---|---|
| 法的書類 v0.3 §6.9 | 「他のAIサービス(Anthropic Claude、OpenAI GPT、xAI Grok、ローカルLLM等)に変更または併用する可能性がある」 |

#### 🟡 論点13: マーケティングメッセージと法務表現の緊張

- 案C: メッセージは断定型のまま使うが、注釈で「※クラッシュ報告など匿名化された技術データの将来導入の余地はあります」のような断り書きを付ける
