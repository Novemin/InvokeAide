### 4.3 例文引用時の引用形式


## 5. ⑤ 設計判断の重複 / 矛盾(Sさん 自主追加)

### 5.1 「禁則」 記述の二重化

| 文書 | 禁則記述 |
|---|---|
| キャラMD §3 / §4 「禁則」 | 「お疲れ様です」 等の事務的敬語は使わない / 説教モード禁止 / 安易な褒め禁止 / 過剰絵文字禁止 |
| コーチングMD §3.2 「Sさん 追加 NG ワード」 | 同上 + 「永続性断定 / 感情主張 / 過剰な気遣い」 を追加 |
| manual.md | 禁則記述なし(ユーザー向けに不要) |

**所見**: 部分的二重化。 ただし **両者は視点が異なる** :
- キャラMD「禁則」 = キャラの性格定義として「こういう言葉は使わない」
- コーチングMD「NG ワード」 = コーチングモード時の特殊文脈で「擬人化誤認 / 依存喚起 を踏まないように」

両方を持っていることで、 通常チャット時は character.md の禁則、 コーチング時は両方適用、 という二重防御になる。 矛盾はなし、 **意図的な重複** 。

**修正提案 F4(運用判断)**:
- 現状維持を推奨(二重防御として機能)
- ただし、 v0.2 改訂時に「キャラMD「禁則」 + コーチング追加分」 という構造で整理する余地あり
- 重大ではない、 軽微な整理タスク
