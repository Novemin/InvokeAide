### F1(v0.2). v0.1 で保留した 5 論点はすべて「v1.2 でも未解消」

v0.1 で「v1.2 本文に既存記述がある可能性」と保留した L-A-5(AI 明示宣言)/ L-A-6(年齢確認)/ L-A-7(解約フロー)/ L-B-10(§15 セキュリティ整合)/ L-C-8(DAO 哲学前置き)の 5 件を v1.2 で確定判定。結果:

- **完全解消 = 0 件**
- **部分的解消 = 1 件**(L-C-8: 哲学は v1.2 §19 で文書化済み、法的書類前置き橋渡し文言は未起草で残作業)
- **確定残存 = 4 件**(L-A-5 / L-A-6 / L-A-7 / L-B-10)

v1.2 は 2026-05-11 起草、法的書類 v0.3 は 2026-05-17 起草。法的書類で新規要求された「AI 明示宣言」「年齢確認」「解約フロー」が仕様書側に取り込まれていないのは時系列的に当然だが、5/26 のスコープ議論で「v1.5 に本文化する」と確定させる必要がある。

### F2(v0.2). v1.2 §15「セキュリティとプライバシー」が「未来の自分を縛らない原則」と直接矛盾(L-A-11)

v1.2 突合せの最大の発見。法的書類 v0.3 §6.5.1 / §8.9 で **避けるべき** と明示された 4 種の断定表現のうち、3 種が v1.2 §15 にそのまま残存している。

| 法的書類が「避ける」表現 | v1.2 §15 該当箇所 |
|---|---|
| 「サーバーは存在しない」 | §15.1「たかしさん側のサーバー: 存在しない」 |
| 「データを一切受信しない」 | §15.5「会話内容を一切受信・保存しない」 |
| 「〜にアクセスできない」 | §15.5「いかなるデータにもアクセスできない」 |

これは v0.1 では L-B-10「§15 整合確認」として抽象論点扱いだったが、v0.2 で具体的な改訂対象として L-A-11 に分解・格上げ(🟡 → 🔴)。仕様書 v1.5 で §15 全面改訂が必要。法的書類 A 案ドラフト本文起草(L-B-8)の前提作業。

### F3(v0.2). v1.2 §19 組織体制図が現行 MIYU 中心、本日の戦略・整理役交代を反映していない(L-A-12)

v1.2 §19.2「戦略・整理役: エルトン」+ 「本番 MIYU: Claude API」+ 「開発エージェント A・B: Claude Code」は、現行 Claude 版 MIYU 中心の体制。InvokeAide 体制(Sさん・Tさん・Uさん)および本日(2026-05-22)の戦略・整理役交代(エルトン → 新担当)が未反映。v1.5 で組織体制図の更新が必要。

これは「指示書 §5 #2 で たかしさんが定義予定の Uさん 標準役割」とも整合させて一括更新するのが効率的。

### F4(v0.2). v1.2 §8.4 セットアップフローの「12 ステップ拡張」が具体的に見えた(L-A-13)

v0.1 では L-A-5 / L-A-6 を「仕様書本文化が必要」と抽象的に記載していたが、v1.2 §8.4 の 10 ステップ構造を確認した結果、具体的な拡張案として「冒頭に AI 明示宣言と年齢確認を 2 ステップ追加し、12 ステップに拡張」が見えた。これは v1.5 改訂時の具体的な実装方針として有用。

### F5(v0.2). v1.2 で「既に確定」している論点が複数あり、5/26 議論を助ける

v0.1 では「v1.4 §27 未確定事項」として記載されていた論点の一部に対し、v1.2 本文に既に方針が書かれていたものがある:

- **L-B-9(価格・課金方式)**: v1.4 §27「買い切り型(または別モデル、要確定)」だが、v1.2 §9.2 で「案1 BYOK 買い切り型(1,200〜2,980円)+ 案2 キャラ追加課金(各 500 円)のハイブリッド」が既に確定済み。v1.4 ドラフトで意図せず後退した可能性、または v1.4 起草時に意図的に再検討に戻された可能性。5/26 で確認。
- **L-C-3(ターゲット層)**: v1.2 §9.1「Urban Hippies の理念に共感する、技術リテラシー中程度の層」「Google AI Studio で API キー発行ができる程度のリテラシー」と明記。CLAUDE.md §1.3 / 法的書類 §3 と整合。「IT 音痴」住み分け論点(L-C-12)も、v1.2 §9.1 を根拠にすれば明文化が容易。
- **L-C-4(マルチキャラ)**: v1.2 §6.3「アプリ初期搭載: ギャル MIYU(標準)、追加キャラ案: 美輪ねえさん風、武士道おじいちゃん、ヒカル風、課金モデル候補: 追加キャラ各 500 円」と方針が明示済み。
- **L-C-13(Apps Script 案)**: v1.2 §4「サーバーレス・アーキテクチャ」+ §11「設計指針」で Apps Script 移植前提のコーディング規約が明示済み。Sさん 技術選定で Apps Script 案が採用された場合、v1.2 §4 / §11 がそのままベースとして機能。

これらは「論点リスト本体」では中立性確保のため「v1.2 突合せで判明した既存記述」(本体 §4.1)として記載し、「5/26 議論時の参照情報」として位置づけた。
