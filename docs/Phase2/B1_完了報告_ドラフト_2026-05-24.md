## 起動して AI 明示モーダルが出るところまで(説明)

### 動作フロー

```
1. ユーザーが ブラウザで http://localhost:5173/ にアクセス
   (本番: PWA インストール / Cloudflare Pages 配信 URL)

2. main.ts が createApp(App) を呼び、 Pinia + Router を install して mount
   - createPinia() でグローバルストア基盤を準備
   - createRouter() で / → HomeView をルート登録

3. App.vue がレンダリング:
   - <h1>InvokeAide</h1>(タイトル、 Tさん 既存テスト維持)
   - <p>秘書召喚アプリ</p>(サブタイトル)
   - <router-view /> → HomeView が表示される
   - <footer class="ai-disclosure">本アプリは AI を用いた対話システムです</footer>
     (常時表示、 Tさん 既存テスト維持)
   - <AiDisclosureModal />(Teleport で <body> 直下に出る)

4. AiDisclosureModal が判定:
   - useDisclosureStore() で needsAcknowledgement を取得
   - LocalStorage の 'invokeaide.disclosure.acknowledged' を確認
   - 値が 'true' でなければモーダル表示(初回起動時)

5. モーダル表示内容(B1 仮文言、 B2 で法的書類 v0.4 確定文言に差し替え):
   - タイトル: 「はじめにご確認ください」
   - 本文 1: 「本アプリは、 AI(人工知能)を用いた対話システムです。」
   - 本文 2: 「応答にはキャラクター演技が含まれますが、 すべて AI が生成しています。
              表示内容の正確性・適切性についてはご自身でご判断ください。」
   - ボタン: 「確認しました」(タップで disclosure.acknowledge() を呼ぶ)

6. 「確認しました」 タップ:
   - LocalStorage に 'invokeaide.disclosure.acknowledged' = 'true' を保存
   - acknowledged.value = true で needsAcknowledgement が false になる
   - v-if が偽になりモーダル消失

7. ユーザーは ホーム画面(HomeView)に到達:
   - <h2>ホーム</h2>
   - <p>ここに、 後の Step で召喚 UI と対話 UI が入ります。</p>
   - footer は常時表示

8. 2回目以降の起動:
   - LocalStorage に 'invokeaide.disclosure.acknowledged' = 'true' があるので
     モーダル非表示、 ホーム画面直接表示
```
