## 1. 注記文案(Sさん 起草)

### 1.1 確定文案(推奨)

```
> このマニュアルでは、 例として「たかし」 というお名前で説明していますが、
> 実際には、 セットアップで設定したあなたのお名前で呼びかけてくれます。
```

### 1.2 文案検討の経緯

整合性チェック §6 のサンプル文案(原案):
> 以下、 例文では便宜的に「たかし」 という名前で示しますが、 実際にはあなたが
> 設定したお名前で呼びかけられます。

Sさん 微修正の意図:
- 「便宜的に」 → 「例として」: 家族テスター層(IT 音痴含む、 [[feedback_ux_three_layer]])を考えると、 「便宜的に」 は公式文書ぽさが出てやや硬い印象。 「例として」 がやわらかい
- 「設定した」 → 「セットアップで設定した」: いつ設定するかが読み手に明確になる(初回セットアップとの紐づけ)
- 「呼びかけられます」 → 「呼びかけてくれます」: 受動形より能動形の方が「キャラが主体的に呼んでくれる」 感触が出る、 manual.md 全体のやわらかいトーンと整合
- 「名前で示しますが」 → 「お名前で説明していますが」: ユーザー名は manual.md 全体で「お名前」 表記なので統一

### 1.3 代替候補(参考)

| # | 文案 | 採否 |
|---|---|---|
| A | 「以下、 例文では便宜的に『たかし』 という名前で示しますが、 実際にはあなたが設定したお名前で呼びかけられます。」 | 整合性チェック §6 のサンプル、 やや硬い印象 |
| B | 「以下では、 例として『たかし』 さんが使う場面を見ていきますが、 実際にはあなたの名前で呼びかけてくれます。」 | やや冗長 |
| **C** | **§1.1 確定文案** | **採用** |
