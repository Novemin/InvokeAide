## 1. 起草の目的

1. 商品化(App Store / Google Play 配布)に向けた最低限の法務基盤の整備
2. BYOK モデル特有の説明責任の明示
3. たかしさんレビュー → 将来の弁護士最終レビュー、という二段構えの起点となる初版を作る

**本書および以後の起草物は、AI/IT法務専門弁護士の最終レビューを商品化前に必ず受けること** を前提とする。エルトンは弁護士ではない。

### 1.1 「ストア審査で落ちない最低限」を優先する

A案のスコープを「完璧法務」ではなく **「実態に合った誠実な説明 + ストア審査通過レベル」** と定義する。これは「手抜き」ではなく「個人〜小規模開発の現実的な優先順位」であり、誠実な姿勢である。

### 1.2 「未来の自分を縛らない」原則(v0.3 新設)

GPT 第2回レビューの最重要指摘:

> 「サーバーゼロ」を断言すると、後で少し機能追加しただけで規約矛盾になる可能性ある。ここは “未来の自分を縛らない” ほうがいい。

A案ドラフト本文の起草時、以下を常に意識する:

- **断定表現を避ける**: 「存在しない」「一切〜しない」のような絶対的表現は、将来の機能追加(クラッシュ分析、エラー収集、Telegram 連携、Push 通知、ライセンス認証等)で矛盾を生む
- **限定表現を使う**: 「専用サーバーを運用していない」「通常運用において収集しない」など、 **何を約束していないかが明確に分かる** 表現
- **誠実さと柔軟性の両立**: ユーザーへの安心感は約束しつつ、将来の正当な機能追加を可能にしておく

これは「曖昧にする」ことではない。「正確に約束する」ことである。

## 6. 各章で必ず盛り込むべき要点

### 6.1 第1章 はじめに

- 本書は利用規約・プライバシーポリシー・免責事項を統合した文書である旨
- **「本アプリは AI を用いた対話システムです」の AI 明示宣言**(EU AI Act 第50条対応)
- 本書への同意がアプリ利用の前提となる旨
- **「Novem Intelligence(以下「運営者」といいます)」の冒頭定義**

### 6.2 第3章 サービス概要

- 「秘書召喚アプリ」とは何か
- **MIYU はキャラクター名、アプリ本体は別** であることの明確化
- AI による応答は人間によるものではないこと
- キャラクター追加・差し替え可能な構造であること

### 6.3 第4章 アカウントとログイン(年齢制限)

- Google アカウントの利用、OAuth 認証
- Gemini API キー(BYOK)の取得・設定
- **年齢制限**:
  - 13歳未満は本サービスを利用できない
  - 利用開始時に年齢確認チェックを行う
  - 虚偽申告について運営者は責任を負わない
  - 13歳〜18歳未満は保護者の同意を得て利用すること

### 6.4 第5章 課金と支払い

- アプリ本体: 買い切り型(または別モデル、要確定)
- **Gemini API: ユーザー本人が Google AI Studio で発行・課金する BYOK モデル**
- 運営者は Gemini API 課金を一切代理しない
- **アプリ価格には Gemini API 利用料は含まれない旨を最大限の強調で明記**
- **返金ポリシー**:
  - App Store / Google Play 経由の購入は各ストアの返金ポリシーに従う
  - 運営者からの直接返金は原則として行わない
  - 「アプリ代金」と「AI 利用料」は別であり、AI 利用料の返金には運営者は関与できない旨

### 6.5 第6章 データの取り扱い(v0.3 で表現を柔軟化)

統合型ゆえ、ここがプライバシーポリシー本体となる。**「未来の自分を縛らない」原則を厳格に適用する**。

#### 6.5.1 GPT指摘を踏まえた表現修正

| v0.2 までの表現(避ける) | v0.3 で採用する表現 |
|---|---|
| 「運営者側サーバーは存在しない」 | 「運営者は、ユーザーの会話内容を保存・閲覧することを目的とした専用サーバーを運用していません」 |
| 「ユーザーデータは一切受信・保存しない」 | 「運営者は、通常運用において、ユーザーの会話内容を収集・保存しません」 |
| 「ユーザーデータにアクセスできない」 | 「運営者は、通常運用において、ユーザーの会話内容にアクセスする手段を持ちません」 |

#### 6.5.2 起草時に含める要素

- ユーザーデータの保管場所: ユーザー自身の Google Drive、端末内
- Gemini API への通信内容には Google の利用規約・プライバシーポリシーが適用される旨
- API キーは端末内に暗号化保存される
- **将来追加される可能性のあるデータ収集** について明示的に余地を残す:
  - クラッシュレポート・エラー解析(匿名化された技術データ)
  - 利用統計(個人を識別しない範囲)
  - ライセンス認証・課金管理に必要なデータ
  - これらを **将来導入する場合は、本規約改訂と事前通知を経る** ことを明記

### 6.6 第7章・第8章 ユーザー生成コンテンツ(v0.3 でファン活動表現を絞り込み)

#### 第7章 ユーザー責任・禁止事項

- 違法な内容を MIYU に保存・記録しないこと
- 他人の個人情報を許諾なく入力しないこと
- 法令違反・公序良俗違反の利用を禁止すること

#### 第8章 知的財産権(v0.3 で表現を絞り込み)

- アプリのコード、UI、ブランドの権利は運営者(または将来の権利者)に帰属
- ユーザーが MIYU との会話を SNS に投稿することは、ユーザー責任において自由
- ただし MIYU のキャラクター画像・音声(VOICEVOX由来)を再配布する際は、各素材のライセンスに従うこと
- **MIYU の二次創作・パロディに関する方針(v0.3 で表現修正)**:
  - 「**非商用かつ常識的範囲のファン活動を歓迎します**」
  - 商用利用、運営者または MIYU のブランドを毀損する利用、ユーザー本人以外の個人を傷つける利用は対象外
  - 詳細なファン活動ガイドラインは別文書として将来整備する可能性がある
- ユーザーが生成したログ・データ自体の権利はユーザーに帰属

**v0.3 のポイント**: 「歓迎」だけだと「許可された」と解釈されるリスクがある(特にAIキャラ系は海外勢の暴走例が多い)。 **「非商用かつ常識的範囲」** という柔らかい限定を明記する。

### 6.7 第9章 免責事項(擬人化誤認)

- AI 回答の正確性は保証しない
- 占い・運勢機能はエンターテイメントであり、医療・法律・財務判断の根拠としない
- 朝サマリの内容の正確性は元情報源に依存
- ユーザーが AI 回答に基づいて行った行為の責任はユーザー自身にある
- AI が誤動作・誤解釈で意図しない予定削除・タスク削除等を行った場合の責任分界
  - → 仕様書第24章の「3択ボタン式確認」が設計上の担保となる旨を関連付ける
- **擬人化誤認への注意**:
  - 「MIYU は AI による生成応答であり、人格・感情・意思を有する実在存在ではありません」
  - キャラクター演技は **会話体験を豊かにする目的の演出** であり、AI が実際に意識・感情を持っていることを意味しない
  - EU AI Act 透明性義務と相性が良い

### 6.8 第10章 AI コンパニオンとしての注意事項

- AI への過度な依存を避けること
- 医療・法律・財務上の重要な判断は専門家に相談すること
- **緊急事態(自殺念慮など)の場合はアプリではなく専門の相談窓口・救急に連絡すること**
- MIYU のキャラ性が時に厳しい言葉として受け取られる可能性への注意喚起

### 6.9 第11章 第三者AIサービスの利用について

- 本サービスは現在 Google 社の Gemini API を利用しているが、 **運営者は技術的・商業的判断に基づき、他のAIサービス(Anthropic Claude、OpenAI GPT、xAI Grok、ローカルLLM等)に変更または併用する可能性がある**
- 利用する AI サービスが変更された場合、運営者はユーザーに対して合理的な事前通知を行う
- 各 AI サービスの利用には、それぞれの提供者の利用規約・プライバシーポリシーが適用される
- ユーザーは利用する AI サービスを変更する際、新たな API キー取得・設定変更が必要となる場合がある

### 6.10 第12章 解約・退会

- ユーザーが Google Drive 内のデータを自分で削除する手順
- アプリ削除と API キー失効の関係
- 「忘れられる権利」への対応(A案では「自己手順での削除可能」の明記のみ、GDPR本格対応はC案)

### 6.11 第14章 サービス終了(v0.3 で責任限定の表現に注意)

- 運営者は以下の事由により、本サービスを終了または一時停止することがある:
  - 上流 AI サービスの料金高騰・仕様変更
  - 運営者の継続困難(健康問題、事業環境変化等)
  - 法改正により本サービスの提供が困難になった場合
  - その他、運営者が合理的に判断する事由
- サービス終了の場合、運営者は可能な限り事前にユーザーに通知する
- サービス終了に伴うユーザー側の損害について、 **運営者の故意または重過失による場合を除き、責任を負わない** (v0.3 で日本法対応を明示)
- サービス終了時、ユーザーデータは Google Drive 内に残るため、ユーザー自身でデータの保存・移行を行うこと

## 8. 留意事項(起草時に頭に入れておくべき論点)

### 8.1 BYOK モデル特有の論点

- 運営者側がデータを保有しない構造により、個人情報保護法上の「個人情報取扱事業者」の典型像から外れる
- ただし「該当しない」と断言するのではなく、「データを受信しない仕組みである」ことを明示する形で、ユーザーの期待形成と実態を一致させる
- ユーザーが API キーを失効・盗難された場合の責任分界

### 8.2 キャラクター AI への依存性配慮

- Replika 事例(2023年伊当局による一時停止命令)の教訓を踏まえ、依存性に関する注意喚起を最低限組み込む
- MIYU のキャラ性が時に厳しい言葉として受け取られる可能性への注意

### 8.3 擬人化誤認への対応

- 「メンタル依存」より「擬人化誤認」のほうが今後の AI 規制で問題化しやすい
- 「実在の人格として接した結果、ユーザーが現実生活で誤った行動を取る」リスク
- 第9章 免責事項に明記、第10章 注意事項とも連動

### 8.4 音声入力誤変換配慮

- 「ユーザー発話を AI が解釈して操作する」構造のため、誤動作リスクを規約で明記
- 削除・送信などの重要操作は確認 UI(3択ボタン式)を経由する仕様を、規約でも担保

### 8.5 占い・運勢機能の位置づけ

- エンターテイメントであることを明記
- 医療判断・人生の重大判断の根拠としない
- オプトイン機能であり、興味のないユーザーには表示されない

### 8.6 「対等な右腕」哲学との両立

- 仕様書第19章にあるように、MIYU は「補佐」ではなく「対等な相棒」として設計されている
- 規約では「サービス提供者-ユーザー」の関係を法的に明確化する必要がある
- 哲学と法務の両立: 前置きで橋渡しする

### 8.7 OSSライセンス管理

- VOICEVOX、npm ライブラリ、UI素材、フォントなどのライセンス整理
- 巻末に「使用しているOSSとライセンス表示」セクションを設ける

### 8.8 責任限定条項と日本法の関係(v0.3 新設)

GPT 第2回レビュー指摘の重要論点。

- 「運営者は責任を負わない」という表現は、日本法では強すぎると **無効化される可能性** がある
- 関連法令: **消費者契約法 第8条**(事業者の損害賠償責任を全部免除する条項の無効)、 **第8条の2**(債務不履行に基づく損害賠償責任の一部免除のうち、故意または重過失によるものを免除する条項の無効)
- 起草時の定型句: **「運営者の故意または重過失による場合を除き」** を必ず付ける
- 商品化前の弁護士レビューで、責任限定条項の具体的文言は最終調整される

A案ドラフト段階では、すべての免責・責任限定の表現に「故意または重過失を除き」のニュアンスを意識的に含める。

### 8.9 「未来の自分を縛らない」原則(v0.3 新設、最重要)

GPT 第2回レビュー指摘の最重要論点。法務文書の本質。

- 法務文書は **完璧さより柔軟性** が重要
- 断定表現(「存在しない」「一切〜しない」「絶対に〜できない」)は、将来の正当な機能追加で矛盾を生む
- 起草時の置換ルール:

| 避ける表現 | 推奨表現 |
|---|---|
| サーバーは存在しない | 専用サーバーを運用していない |
| データを一切受信しない | 通常運用において収集・保存しない |
| 〜にアクセスできない | 〜にアクセスする手段を持たない |
| 100% 安全 | 合理的なセキュリティ対策を講じる |
| 永久に〜しない | 現時点では〜していない |

- これは「曖昧にする」ことではない。 **「正確に約束する」** ことである
- 「何を約束し、何を約束していないか」が明確になることで、ユーザーへの誠実さと運営者の柔軟性が両立する
