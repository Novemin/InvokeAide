## 2026-05-18 朝 たかしさん統一返答で確定した運用方針(テスト戦略 §10 / ドキュメント整合 §8)

### AI 明示宣言の表示(§10 #1)
- **画面の一番下に常に小さく表示**(全画面)
- **初回オンボーディングで大きく表示**
- 設定 → 「このアプリについて」で常時参照可能
- 毎発言ごとの注記は不要
- EU AI Act §50 / 法的書類 v0.3 §6.1 要件はこの構成で充足

### 擬人化誤認注記の表示位置(§10 #2 / §8.4)
**3箇所に表示**:
1. 初回オンボーディング(AI 明示宣言と一緒に)
2. キャラクター選択画面の脚注(常時見える)
3. 設定 → 「このアプリについて」(参照したい時)
- 仕様書 v1.5 §6 に「キャラクター演技の位置づけ」節を新設、表示要件・位置・文言を規定
- 仕様書 v1.5 §22 入力UX 本文化時に AI 明示と擬人化誤認の両方を UI 設計レベルで確定
- 3択ボタン式確認との統合: 誤動作時の確認画面でも擬人化誤認を念押し

### テスト用 Google アカウント(§10 #3)
- たかしさん主管で **取得済み**(Novem Intelligence 名義)
- 認証情報管理: GitHub Secrets
- 将来のサーバー側運営(クラッシュレポート等)にも資産として活用

### 家族テストフィードバック収集方式(§10 #4)
**LINE/メール経由のアンケート形式に確定**(Tさん 感触の「完全口頭」から修正):
- アプリ側ログ収集: なし(法的書類 v0.3 §6.5 変更不要)
- フィードバック収集チャネル: LINE / メール(アンケート形式)
- アンケート: Google Forms 等で質問一覧準備
- 収集頻度:
  - ベータ期間最初の2週間: 1週間ごと(初期バグ・UX 問題早期キャッチ)
  - その後: 月1回(運用安定後、長期傾向把握)
- 不明点: 随時、LINE / メール
- 質問項目案: 「今週よく使った機能」「困った点」「もっと欲しい機能」「MIYU との会話で印象的だった瞬間」等
- 質問一覧の起草: たかしさん主導、エルトン整理サポート(家族配布前の別タスク)、Tさん が UX テスト戦略観点でレビュー予定

### CI 環境(§10 #5)
- **GitHub + GitHub Actions + プライベートリポジトリ** 確定
- 引継ぎ帳「GitHub push 禁止」懸念は Phase 0 デモのハードコード問題が原因、新規構築 + BYOK 設計画面方式なら認証情報がコードに混入しない構造でリスクなし
- リポジトリ: GitHub プライベートで開始、商品化フェーズで公開検討
- CI/CD: GitHub Actions(PR時テスト + マージ後 E2E + 毎日クロスブラウザ)
- 認証情報管理: GitHub Secrets
- 将来 GPT/Codex や他 AI エージェントが参加する基盤として GitHub 中心を歓迎

### マーケコピーと法務表現の分離(§8.1)
- マーケコピー(外向き、ストア・公式サイト・SNS): 訴求力重視、注釈付き
- アプリ内表記(ユーザーが見る画面): **法的書類 v0.3 §1.2 限定型に統一**
- 利用規約・プライバシーポリシー: 限定型に統一
- 仕様書 v1.5 §9.4 マーケコピーは「外向け表現の見本」として残しつつ、§15 本文・実装・アプリ内表記は限定型で統一
