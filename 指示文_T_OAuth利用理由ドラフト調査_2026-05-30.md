## 2. やってほしいこと

### 2-1. 各スコープの「使い方シーン」を整理

以下のスコープそれぞれについて、**InvokeAide で具体的にいつ・どのような操作で使うか**を箇条書きで整理してください。これは justification の素材になります。

| スコープ | 整理項目 |
|---|---|
| `openid` | ユーザー識別、認証セッション維持 |
| `email` | ユーザー識別、サポート連絡先取得 |
| `profile` | 表示名・アバター表示（任意） |
| `https://www.googleapis.com/auth/drive.file` | データ保存・読み出し、`MIYU_App_Data/` 配下のファイル管理 |
| `https://www.googleapis.com/auth/tasks` | ユーザーのタスクの確認・追加・更新・完了・削除 |
| `https://www.googleapis.com/auth/calendar.events` | ユーザーの予定の確認・追加・更新・削除 |

各スコープについて、以下を整理:

**a. 操作シーン**: 具体的にユーザーがどんな入力をした時にこのスコープが使われるか  
（例: 「ユーザーが『明日10時に歯医者の予定入れて』と発話した時、`calendar.events` でメインカレンダーに予定を作成」）

**b. データの流れ**: アプリ → Google API への呼び出し、レスポンスの扱い  
（例: 「calendar.events.insert API を呼ぶ。レスポンスの event ID は会話ログに紐づけて Drive 保存」）

**c. データの保存先**: Google のサーバーに保存されるのか、ユーザー Drive か、端末内か  
（例: 「予定は Google カレンダー上に保存される（運営者は保存しない）」）

**d. なぜこのスコープが本質的に必要か**: 代替手段がない理由  
（例: 「予定の追加・編集機能は本アプリの中核機能。`calendar.events` がないと実現不可能」）
