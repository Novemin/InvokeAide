## 0. 背景（なぜ消すか）

Calendar スコープが `https://www.googleapis.com/auth/calendar.events` に確定し、「**専用サブカレンダーは作らず、ユーザーのメインカレンダーに集約する**」方針になった（5/28 確定、5/30 再確認）。

これに伴い、専用カレンダーを前提にした型が不要になった。`src/interfaces/domain.ts` の以下2つを削除する:

- `dedicatedCalendarId`
- `manageMainCalendar`

---

## 1. 進め方（調査 → 削除 → ビルド確認の順）

**いきなり消さず、まず参照箇所を洗い出すこと**（消したあとに他の箇所がビルドエラーになるのを防ぐため）。

### ステップ1：調査
1. `src/interfaces/domain.ts` を開き、`dedicatedCalendarId` / `manageMainCalendar` の**定義箇所**を確認する。
2. リポジトリ全体で、この2つの型を**参照している箇所**を洗い出す（例: `grep -rn "dedicatedCalendarId\|manageMainCalendar" src/`）。
3. 参照箇所の一覧を、削除前に把握しておく。

### ステップ2：削除と整理
4. `domain.ts` から当該2つの型定義を削除する。
5. ステップ1で見つかった**参照箇所も、専用カレンダーを作らない方針に沿って整理する**:
   - 単に専用カレンダー用途で使われているだけなら、その参照も削除
   - 他のロジックと絡んでいて単純削除できない箇所があれば、**消さずに報告**（勝手に大きな変更をしない）

### ステップ3：ビルド確認
6. `npm run typecheck` と `npm run build` が**通ること**を確認する。型を消したことで壊れた箇所がないか必ず確認。

---

## 2. commit / push

- **1 commit 1 トピック**を守る（この型削除だけで1 commit）。
- **自分が触ったファイルのみ** `git add` する（他者の変更を巻き込まない）。
- commit メッセージ例: `refactor: remove dedicatedCalendarId / manageMainCalendar (calendar.events に集約)`
- push 前に `git status` でステージ内容を確認。
